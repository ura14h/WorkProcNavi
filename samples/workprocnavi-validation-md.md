# WorkProcNavi 機能確認手順（Markdown 単体版）

この手順書は、単一の Markdown ファイルを読み込んだ場合に、概要表示、フェーズ進行、コードコピー、手順本文内リンク、セッション保存、再開、エビデンス出力が設計どおりに動作するかを確認するためのサンプルである。

## 取り込みと概要確認

最初に、単体 Markdown の取り込みと概要表示を確認する。

### 手順書を読み込む

アプリを起動し、このファイルをホーム画面へドラッグ＆ドロップする。

- [ ] ホーム画面に `.md` / `.zip` / セッションファイル用のドロップ領域が表示されている
- [ ] `workprocnavi-validation-md.md` をドロップするとエラーなく読み込まれる
- [ ] 読み込み後に概要画面へ遷移する

### 概要を確認して開始する

概要画面に表示されたタイトル、フェーズ数、ステップ数を確認してから作業を開始する。

- [ ] 手順書タイトルが `WorkProcNavi 機能確認手順（Markdown 単体版）` と表示される
- [ ] フェーズ数とステップ数が手順書の構成と一致している
- [ ] 開始操作で実行画面へ遷移する

## 実行中の操作確認

画面上のガイド表示、コードブロック、確認項目、フェーズ移動を確認する。

### コードブロックとステップ移動を確認する

次のコマンド例を使い、コードブロックの表示とコピー動作を確認する。

```bash
echo "WorkProcNavi copy test"
```

- [ ] コードブロックが整形表示され、コピーボタンが見える
- [ ] コピーした内容を任意のエディタへ貼り付けると、コードブロックの内容と一致する
- [ ] `前のステップ` / `次のステップ` の操作で現在ステップ位置を移動できる

### 確認項目とフェーズ遷移を確認する

このステップでは、確認項目の保持と次フェーズへの進み方を確認する。

- [ ] 確認項目を 1 件ずつチェックすると状態が即時に反映される
- [ ] 未確認項目を残したままフェーズ移動を試すと、設計どおりの制御または確認が行われる
- [ ] このフェーズ内の確認項目を完了すると次フェーズへ進める

### 手順本文内リンクを確認する

外部サイト URL は [Example Domain](https://example.com/) を使って確認する。

`file://` の確認では、フォルダリンクは実在する場所を使う。ファイルリンクは、対象ファイルが存在する場合と存在しない場合の両方を確認する。

- macOS / Linux フォルダ例: [一時フォルダ](file:///tmp)
- macOS / Linux ファイル例: [一時ファイル](file:///tmp/workprocnavi-link-open-sample.txt)
- macOS / Linux 未存在ファイル例: [存在しない一時ファイル](file:///tmp/workprocnavi-link-open-missing-sample.txt)
- Windows フォルダ例: [一時フォルダ](file:///C:/Temp)
- Windows ファイル例: [一時ファイル](file:///C:/Temp/workprocnavi-link-open-sample.txt)
- Windows 未存在ファイル例: [存在しない一時ファイル](file:///C:/Temp/workprocnavi-link-open-missing-sample.txt)

- [ ] 外部サイト URL をクリックすると、アプリ内ではなく既定の外部ブラウザで開く
- [ ] `file://` フォルダリンクをクリックすると、標準ファイルエクスプローラまたは Finder でフォルダが開く
- [ ] `file://` ファイルリンクをクリックしてもファイル本体は直接開かず、親フォルダ内で対象ファイルが表示される
- [ ] 存在しない `file://` ファイルリンクをクリックすると、ファイルが見つからない旨と親フォルダを開くかの確認が表示される
- [ ] 上記確認で続行すると、対象ファイルは開かず親フォルダだけが開く
- [ ] リンクを開いてもフェーズ位置、チェック状態、画面表示は維持される

## 保存・再開・完了確認

最後に、セッション保存、再開、完了後のエビデンス出力を確認する。

### セッション保存と再開を確認する

途中で保存または中断を行い、作業の再開可否を確認する。

- [ ] `.session` ファイルが元の Markdown ファイルと同じ場所に作成される
- [ ] アプリを再起動して `.session` をドロップすると、前回のフェーズ位置とチェック状態が復元される
- [ ] 再開後も残りの確認項目を続けて操作できる

### エビデンス出力を確認する

最終フェーズ完了後の出力結果を確認する。

- [ ] 完了時にサマリーまたは完了画面が表示される
- [ ] 元ファイルと同じ場所に `.log` ファイルが出力される
- [ ] `.log` に開始日時、完了日時、手順書名、各確認項目の結果が記録される
